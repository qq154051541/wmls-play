## 一、前言

本人最近刷很多做手势互动结合3D粒子特效视频，以及forever组合其他成员模仿阿信的招牌手势后突发灵感，觉得可以结合起来做个手势互动页面，没想到还是挺受欢迎。由于个人日常生活和工作精力问题，前后累积花起来也就分别花一个晚上和一个下午的时间去做了这两个页面和录制发布。因此还存在很多细节需要打磨，希望有兴趣的wmls可以继续打磨或借鉴做得更好。也正是因为时间关系，恋爱ing中的汽车我偷个懒，由ai生成照片，然后去除背景用照片形式引入。

现在都是AI辅助写代码的时代，因为门槛也比较低，这俩个页面也基本都是AI写出来，大家也可以继续通过你的描述，让AI帮你写，或者你把界面上传到一些AI平台，让它帮你优化，再复制过来运行。

考虑到大多数wmls并不是技术出身，因此开源后可以直接挂载访问或者自己按步骤本地开服务运行。此工具只作为个人兴趣爱好和学习使用，勿商业化哦~

## 二、使用前需要准备什么？

### 1. 设备要求

请确认具备以下条件：

* 💻 一台电脑（Windows 或 Mac 都可以）
* 📷 一个可以正常使用的摄像头

  * 笔记本自带摄像头即可
  * 台式机需要外接摄像头
* 🌐 Chrome 浏览器（强烈推荐）或其他主流较新版本的浏览器均可

---

### 2. 使用环境建议

为了效果更好，请尽量做到：

* 人站在摄像头前 左下角会看到你自己的画面（底部有个按钮点击可以显示隐藏画面）
* 手部不要被遮挡
* 环境光线充足（不要太暗）
* 背景尽量简单

---


## 三、页面打开后你会看到什么？

### 1️⃣ 加载画面

* 页面中间显示：
  **LOADING VISION ENGINE…**
* 稍等几秒即可

---

### 2️⃣ 摄像头画面出现

* 左下角会看到你自己的画面（镜像）
* 这是系统在“看你的手”

如果不想看到自己：

* 点击下方按钮
  👉 **HIDE CAMERA**
* 不影响互动效果

## 五、本地如何运行这个页面（非常重要）

除了线上访问，本地自己想改造运行也很简单，这是一个 **纯静态网页**，需要开启服务就可运行。

### 5.1代码编辑器启动

有些浏览器在**直接双击 HTML 文件**时，会限制摄像头或本地资源。

通过 VSCode、cursor或trae等编程软件，启动一个本地服务即可。或者有技术大佬本地有比如像Node等服务也是可以。

### 5.2 准备工作

#### 第一步：安装 编程软件VSCode、cursor或trae

我是建议按照trae，因为有免费额度，当然有很多懂行的技术大佬都会用cursor。其实都一样，我们这里介绍的是面向普通的大众如何运行即可。你也可以上传代码在AI对话让AI帮你修改再粘贴过来运行。

---

#### 第二步：安装「Live Server」插件

1. 打开插件管理
2. 左侧点击 **扩展图标**（像四个小方块）
3. 在搜索框输入：

   ```
   Live Server
   ```
4. 找到作者是 **Ritwick Dey** 的插件
5. 点击 **Install（安装）**

👉 只需要安装一次

---

### 5.3 用 Live Server 启动页面（核心步骤）

#### 第一步：打开项目文件夹

1. 打开 VS Code
2. 点击菜单：

   * **File → Open Folder**
3. 选择你存放页面的那个文件夹
   （里面能看到 `index.html` 或主 HTML 文件）

---

#### 第二步：启动本地服务

方式一（推荐）：

1. 在编程软件中
   **右键 对应的html文件**
2. 点击：
   👉 **Open with Live Server**

方式二：

* 打开对应的html文件，双击直接点击 VS Code 右下角
  👉 **Go Live**

---

#### 第三步：浏览器自动打开

* 浏览器会自动打开
* 地址类似：

  ```
  http://127.0.0.1:5500/index.html
  ```
* 页面正常显示后：

  * 浏览器弹出摄像头权限
  * 点击：✅ **允许**

---


## 6、如果出问题了怎么办？

### 看不到画面变化？

* 检查摄像头是否被挡住
* 确认已经允许摄像头权限
* 换个光线亮一点的地方

---

### 页面显示 CAMERA ERROR？

* 关闭页面
* 重新用 Chrome 打开
* 再次允许摄像头

---

### 界面卡顿

* 界面如果卡顿可能是电脑跑不掉或者开太多窗口，可只保留一个页面经常刷新看看。
* 若为了拍摄视频后期可以利用剪辑软件进行调整
